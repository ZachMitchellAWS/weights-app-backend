# Checkin Service

Checkin service endpoints for managing exercise check-ins, lift sets, and estimated 1RM records.

---

## Exercises

All exercise endpoints use the plural `/checkin/exercises` path and support batch operations.

### POST /checkin/exercises (Upsert)
Creates new exercises or updates existing ones.

**Auth:** API Key + Access Token

**Upsert Behavior:**
| Scenario | Behavior |
|----------|----------|
| New item | Creates with all fields including `createdDatetime` |
| Existing item | Updates `name`, `isCustom`, `loadType`, `notes`; preserves `createdDatetime`, `createdTimezone` |

**Request:**
```json
{
  "exercises": [
    {
      "exerciseItemId": "uuid",
      "name": "Bench Press",
      "isCustom": false,
      "loadType": "Barbell",
      "createdTimezone": "America/Los_Angeles",
      "createdDatetime": "2026-01-27T10:30:00.000Z",
      "notes": "Optional notes",
      "icon": "figure.strengthtraining.traditional"
    }
  ]
}
```

**Exercise Fields:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `exerciseItemId` | string (UUID) | Yes | Unique identifier generated by frontend |
| `name` | string | Yes | Exercise name |
| `isCustom` | boolean | Yes | Whether exercise is user-created |
| `loadType` | string | Yes | One of: `"Barbell"`, `"Single Load"` |
| `createdTimezone` | string | Yes | IANA timezone (e.g., `"America/Los_Angeles"`) |
| `createdDatetime` | string | Yes | ISO 8601 datetime when created (frontend-supplied) |
| `notes` | string | No | Optional notes |
| `icon` | string | No | Icon identifier (SF Symbol or custom asset name) |

**Response (200 for updates, 201 for pure creates):**
```json
{
  "exercises": [...],
  "created": 1,
  "updated": 0
}
```

---

### GET /checkin/exercises
Retrieves all non-deleted exercises for the authenticated user.

**Auth:** API Key + Access Token

**Response (200):**
```json
{
  "exercises": [
    {
      "userId": "uuid",
      "exerciseItemId": "uuid",
      "name": "Bench Press",
      "isCustom": false,
      "loadType": "Barbell",
      "createdTimezone": "America/Los_Angeles",
      "notes": "Felt strong today",
      "icon": "figure.strengthtraining.traditional",
      "createdDatetime": "2026-01-27T10:30:00.000Z",
      "lastModifiedDatetime": "2026-01-27T10:30:00.000Z"
    }
  ]
}
```

**Filtering:**
- Only returns exercises where `deleted` is not `true`
- Sorted by `exerciseItemId`

---

### DELETE /checkin/exercises (Batch)
Soft deletes exercises by setting `deleted=true`.

**Auth:** API Key + Access Token

**Request:**
```json
{
  "exerciseItemIds": ["uuid-1", "uuid-2"]
}
```

**Response (200):**
```json
{
  "message": "Deleted 2 exercise(s)",
  "deletedExercises": [...],
  "notFoundIds": []
}
```

**Behavior:**
- Sets `deleted=true` and updates `lastModifiedDatetime`
- Data preserved in database (soft delete)
- Deleted items excluded from GET responses
- Returns `notFoundIds` for any IDs not found for this user

## Lift Sets

Lift sets represent individual sets of an exercise (reps + weight). All endpoints use `/checkin/lift-sets` and support batch operations.

### POST /checkin/lift-sets (Batch Create)
Creates one or more lift sets.

**Auth:** API Key + Access Token

**Request:**
```json
{
  "liftSets": [
    {
      "liftSetId": "uuid",
      "exerciseId": "uuid-of-exercise",
      "reps": 10,
      "weight": 135.5,
      "createdTimezone": "America/Los_Angeles",
      "createdDatetime": "2026-01-27T10:30:00.000Z"
    }
  ]
}
```

**Lift Set Fields:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `liftSetId` | string (UUID) | Yes | Unique identifier generated by frontend |
| `exerciseId` | string (UUID) | Yes | ID of the associated exercise |
| `reps` | integer | Yes | Number of repetitions (non-negative) |
| `weight` | number | Yes | Weight used (decimal allowed) |
| `createdTimezone` | string | Yes | IANA timezone (e.g., `"America/Los_Angeles"`) |
| `createdDatetime` | string | Yes | ISO 8601 datetime when created (frontend-supplied) |

**Response (201 Created):**
```json
{
  "liftSets": [
    {
      "userId": "uuid",
      "liftSetId": "uuid",
      "exerciseId": "uuid",
      "reps": 10,
      "weight": 135.5,
      "createdTimezone": "America/Los_Angeles",
      "createdDatetime": "2026-01-27T10:30:00.000Z",
      "lastModifiedDatetime": "2026-01-27T10:30:00.000Z"
    }
  ],
  "created": 1
}
```

---

### GET /checkin/lift-sets (Paginated)
Retrieves paginated lift sets, ordered by most recent first.

**Auth:** API Key + Access Token

**Query Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | integer | 100 | Items per page (max 500) |
| `pageToken` | string | - | Token from previous response for next page |

**Response (200):**
```json
{
  "liftSets": [...],
  "count": 100,
  "hasMore": true,
  "nextPageToken": "base64-encoded-token"
}
```

**Pagination Flow:**
1. Call without `pageToken` to get most recent items
2. If `hasMore` is `true`, call again with `pageToken=nextPageToken`
3. Repeat until `hasMore` is `false`

**Sync Strategy:**
- First page contains most recent data for immediate use
- Subsequent pages bring historical data into sync
- Soft-deleted items are automatically filtered out

---

### DELETE /checkin/lift-sets (Batch)
Soft deletes lift sets by setting `deleted=true`.

**Auth:** API Key + Access Token

**Request:**
```json
{
  "liftSetIds": ["uuid-1", "uuid-2"]
}
```

**Response (200):**
```json
{
  "message": "Deleted 2 lift set(s)",
  "deletedLiftSets": [...],
  "notFoundIds": []
}
```

**Behavior:**
- Sets `deleted=true` and updates `lastModifiedDatetime`
- Data preserved in database (soft delete)
- Deleted items excluded from GET responses
- Returns `notFoundIds` for any IDs not found for this user

## Estimated 1RM

Estimated 1RM records track personal records achieved during lift sets. All endpoints use `/checkin/estimated-1rm` and support batch operations.

### POST /checkin/estimated-1rm (Batch Create)
Creates one or more estimated 1RM records.

**Auth:** API Key + Access Token

**Request:**
```json
{
  "estimated1RMs": [
    {
      "estimated1RMId": "uuid",
      "liftSetId": "uuid-of-lift-set",
      "exerciseId": "uuid-of-exercise",
      "value": 225.5,
      "createdTimezone": "America/Los_Angeles",
      "createdDatetime": "2026-01-27T10:30:00.000Z"
    }
  ]
}
```

**Estimated 1RM Fields:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `estimated1RMId` | string (UUID) | Yes | Unique identifier generated by frontend |
| `liftSetId` | string (UUID) | Yes | ID of the lift set that achieved this E1RM |
| `exerciseId` | string (UUID) | Yes | ID of the associated exercise |
| `value` | number | Yes | The estimated 1RM value (decimal allowed) |
| `createdTimezone` | string | Yes | IANA timezone (e.g., `"America/Los_Angeles"`) |
| `createdDatetime` | string | Yes | ISO 8601 datetime when created (frontend-supplied) |

**Response (201 Created):**
```json
{
  "estimated1RMs": [
    {
      "userId": "uuid",
      "liftSetId": "uuid",
      "estimated1RMId": "uuid",
      "exerciseId": "uuid",
      "value": 225.5,
      "createdTimezone": "America/Los_Angeles",
      "createdDatetime": "2026-01-27T10:30:00.000Z",
      "lastModifiedDatetime": "2026-01-27T10:30:00.000Z"

  ],
  "created": 1
}
```

## Recovery Checkins

Recovery checkins capture daily subjective readiness data. All endpoints use `/checkin/recovery-checkins` and support batch operations.

### POST /checkin/recovery-checkins (Batch Create)
Creates one or more recovery checkins.

**Auth:** API Key + Access Token

**Request:**
```json
{
  "checkins": [
    {
      "recoveryCheckinId": "uuid",
      "checkinDate": "2026-03-16",
      "primaryResponse": "ready",
      "severityLevel": null,
      "planningToTrain": null,
      "createdTimezone": "America/Los_Angeles",
      "createdDatetime": "2026-03-16T10:30:00.000Z"
    }
  ]
}
```

**Recovery Checkin Fields:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `recoveryCheckinId` | string (UUID) | Yes | Unique identifier generated by frontend |
| `checkinDate` | string | Yes | "YYYY-MM-DD" — the day the response is for |
| `primaryResponse` | string | Yes | One of: `ready`, `good`, `slightly_fatigued`, `very_fatigued`, `sick` |
| `severityLevel` | string | No | For `sick`: `mild`, `moderate`, `severe` |
| `planningToTrain` | boolean | No | For `very_fatigued`/`sick` |
| `createdTimezone` | string | Yes | IANA timezone (e.g., `"America/Los_Angeles"`) |
| `createdDatetime` | string | Yes | ISO 8601 datetime when created (frontend-supplied) |

**Response (201 Created):**
```json
{
  "checkins": [...],
  "created": 1
}
```

---

### GET /checkin/recovery-checkins (Paginated)
Retrieves paginated recovery checkins, ordered by checkinDate descending.

**Auth:** API Key + Access Token

**Query Parameters:**
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | integer | 2000 | Items per page (max 2000) |
| `pageToken` | string | - | Token from previous response for next page |
| `startDate` | string | - | Optional YYYY-MM-DD filter (inclusive) |
| `endDate` | string | - | Optional YYYY-MM-DD filter (inclusive) |

**Response (200):**
```json
{
  "checkins": [...],
  "count": 14,
  "hasMore": false
}
```

---

## Groups

Named, ordered collections of exercise UUIDs. Used to define which exercises appear in the check-in view's horizontal selector.

### POST /checkin/groups (Upsert)
Creates new groups or updates existing ones.

**Auth:** API Key + Access Token

**Upsert Behavior:**
| Scenario | Behavior |
|----------|----------|
| New item | Creates with all fields including `createdDatetime` |
| Existing item | Updates `name`, `exerciseIds`, `isCustom`, `sortOrder`; preserves `createdDatetime`, `createdTimezone` |

**Request:**
```json
{
  "groups": [
    {
      "groupId": "uuid",
      "name": "Tier Exercises",
      "exerciseIds": ["uuid-1", "uuid-2", "uuid-3"],
      "isCustom": false,
      "sortOrder": 0,
      "createdTimezone": "America/Los_Angeles",
      "createdDatetime": "2026-03-17T10:30:00.000Z"
    }
  ]
}
```

**Group Fields:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `groupId` | string (UUID) | Yes | Unique identifier generated by frontend |
| `name` | string | Yes | Group name |
| `exerciseIds` | string[] (UUIDs) | Yes | Ordered list of exercise IDs in this group |
| `isCustom` | boolean | Yes | Whether group is user-created |
| `sortOrder` | integer | Yes | Display order |
| `createdTimezone` | string | Yes | IANA timezone (e.g., `"America/Los_Angeles"`) |
| `createdDatetime` | string | Yes | ISO 8601 datetime when created (frontend-supplied) |

**Response (200 for updates, 201 for pure creates):**
```json
{
  "groups": [...],
  "created": 1,
  "updated": 0
}
```

---
